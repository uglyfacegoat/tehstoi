# Responsive Breakpoints

## Контрольные размеры

| Группа | Viewport | Назначение |
| --- | ---: | --- |
| Mobile XS | 320x568 | маленькие iPhone/Android |
| Mobile S | 360x640 | базовые Android |
| Mobile M | 390x844 | iPhone 12/13/14 |
| Mobile L | 430x932 | крупные iPhone/Android |
| Tablet portrait | 768x1024 | iPad / планшеты вертикально |
| Tablet landscape | 1024x768 | iPad / планшеты горизонтально |
| Laptop small | 1280x720 | компактные ноутбуки |
| Laptop common | 1366x768 | частый ноутбучный размер |
| Laptop wide | 1440x900 | макеты и ноутбуки 14-15" |
| Desktop 1080p | 1920x1080 | обычный монитор |
| Desktop 2K/QHD | 2560x1440 | 2K монитор |
| Desktop 4K | 3840x2160 | 4K монитор |

## CSS-режимы

- `>= 1800px`: десктопный макет масштабируется крупнее для 1080p/2K/4K.
- `1200-1439px`: сохраняется desktop-композиция 1440px с аккуратным масштабированием.
- `901-1199px`: tablet/laptop compact, без глобального scale, секции становятся проще.
- `<= 900px`: tablet/mobile, крупные сетки складываются в одну колонку.
- `<= 640px`: телефон, декоративные или сложные ассеты скрываются либо упрощаются.
- `<= 420px`: маленький телефон, уменьшаются заголовки, кнопки и внутренние отступы.

## Правило для сложных блоков

Если декоративная схема, плашка или фотография на телефоне начинает перекрывать текст, она скрывается или переводится в обычный статичный блок. Приоритет: читаемый текст, кликабельные кнопки и отсутствие горизонтального скролла.
